# PROJECT OVERVIEW

Building a distributed heterogeneous training system for ResNet on Tiny ImageNet-200.
Multiple laptops (Mac MPS + Windows CUDA) connected via Tailscale VPN.
Architecture: Parameter Server (NOT PyTorch DDP). Leader holds model, workers compute
gradients, push compressed gradients to leader, leader aggregates + steps optimizer,
returns weight delta to workers.

Why not DDP: MPS and CUDA cannot share NCCL/Gloo backends. DDP requires homogeneous
hardware and serializes on slowest worker. Parameter server handles heterogeneous devices
and unequal batch sizes naturally.

# TECHNICAL ARCHITECTURE

## Communication
- Transport: gRPC over Tailscale WireGuard VPN
- Leader port: 50051 (default)
- Protocol: protobuf (proto/trainer.proto)
- Message size limit: 256 MB on both server (leader.py) and client channels (worker.py)
- Leader disables grpc.so_reuseport to prevent multiple leader processes sharing 50051
- Heartbeat interval: 5 seconds (asyncio.sleep(5) in worker heartbeat_loop)
- Worker auto-reconnect after leader restart: 10 second retry interval

## gRPC Channel Design (worker)
- async_ch = aio.insecure_channel(...) → for Heartbeat (bidi stream) + GetAssignment (blocking unary)
- sync_ch = grpc.insecure_channel(...) → for ExchangeGradients (called from training thread)
- Reason: training loop runs in asyncio.get_event_loop().run_in_executor(), can't use aio there

## Leader internals
- asyncio.Lock (_lock): guards _workers dict
- asyncio.Condition (_grad_cond): gradient sync — last worker to arrive aggregates + notifies all
- asyncio.Event (_training_started): set when operator types 'start' or auto-start fires
- Per-worker asyncio.Queue (assignment_queue): start_training() puts sentinel, unblocks GetAssignment
- heartbeat_monitor(): removes timed-out workers, handles dead-worker mid-sync
- _grad_arrival dict + _prev_round_end: per-round latency tracking for straggler detection
- _count_alive(): only counts workers with assigned=True (late joiners excluded from grad sync
  until they complete GetAssignment and start training)

## Data flow per training step
1. Worker: forward pass → loss.backward() → compress_gradients() [Top-K per layer]
2. Worker: ExchangeGradients RPC → sends GradientPush (worker_id, step, loss, sparse_grads)
   Leader records arrival time per worker in _grad_arrival
3. Leader: collects from all alive+assigned workers (asyncio.Condition wait loop)
4. Leader: last worker triggers _aggregate_and_step() → weighted average → optimizer.step()
   Computes per-worker compute_ms and wait_ms, logs straggler if wait > 200ms
5. Leader: serializes weight delta (new_weights - old_weights) → sends WeightUpdate
   Logs round_ms to RunRecorder.log_step()
6. Worker: apply_delta() → adds delta to model params in-place

## Gradient compression
Top-K sparsification: for each layer, keep topk_k elements by absolute value.
Sends (indices, values, shape) tuples. Default topk_k=50000. topk_k=0 = full gradients.

## Sharding
worker_fraction = worker_score / sum(all_scores)
worker_samples = int(fraction × 100,000)  [Tiny ImageNet train size]
batch_size scales proportionally (base from --batch-size flag, default 32, min 8, max 256)
Late joiners: random sample from full dataset (may overlap — acceptable for PS)

## RunRecorder output
runs/<model>__<dataset>__<optimizer>__topk<K>__<timestamp>/
  config.json     hyperparameters snapshot
  metrics.csv     columns: step, loss, round_ms, epoch, val_acc
  summary.json    final_loss, final_val_acc, total_steps, epoch_accs
  loss.png        loss curve (requires matplotlib)

# KEY DECISIONS & CONSTRAINTS

1. Parameter Server over DDP: MPS + CUDA can't use NCCL. PS allows heterogeneous hardware.

2. Two gRPC channels per worker (async + sync): training thread uses run_in_executor.

3. Weight delta (not full state dict) per step: reduces bandwidth from ~170 MB to ~1-10 MB.

4. asyncio.Condition for gradient sync: "last worker aggregates" semantics.

5. grpc.aio has NO sleep() — use asyncio.sleep() in async contexts.

6. Leader gRPC server needs explicit 256 MB message size options (default 4 MB rejects grads).

7. Worker self-bootstrap: _bootstrap() uses only stdlib, runs before third-party imports,
   creates .venv, installs deps, generates proto stubs, re-execs via os.execv on Mac/Linux.
   Deps skip: MD5 hash of requirements.txt stored in .venv/.deps_hash; pip skipped if match.

8. Late worker joining: accepted mid-run, gets proportional random shard + current model weights.
   _count_alive() excludes unassigned workers so they don't block existing gradient rounds.

9. Auto-reconnect: leader_disconnected flag in WorkerSharedState distinguishes gRPC failure
   from explicit STOP command. On disconnect: sleep 10s → re-register with new leader.

10. Straggler detection: logged but not mitigated (out of scope for current 3-experiment plan).
    grad_sync_timeout (120s) handles extreme stalls. Per-worker latency visible in logs + CSV.
