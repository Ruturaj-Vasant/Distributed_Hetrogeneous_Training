# PROJECT OVERVIEW

Building a distributed heterogeneous training system for ResNet on Tiny ImageNet-200.
Multiple laptops (Mac MPS + Windows CUDA) connected via Tailscale VPN.
Architecture: Parameter Server (NOT PyTorch DDP). Leader holds model, workers compute
gradients, push compressed gradients to leader, leader aggregates + steps optimizer,
returns weight delta to workers.

Why not DDP: MPS and CUDA cannot share NCCL/Gloo backends. DDP requires homogeneous
hardware and serializes on slowest worker. Parameter server handles heterogeneous devices
and unequal batch sizes naturally.

# TECHNICAL ARCHITECTURE

## Communication
- Transport: gRPC over Tailscale WireGuard VPN
- Leader port: 50051 (default)
- Protocol: protobuf (proto/trainer.proto)
- Message size limit: 256 MB on both server and client channels
- Leader disables grpc.so_reuseport to prevent multiple leader processes on 50051
- Heartbeat interval: 5 seconds
- Worker auto-reconnect after leader restart: 10 second retry interval

## gRPC Channel Design (worker)
- async_ch = aio.insecure_channel(...)  → Heartbeat (bidi stream) + GetAssignment (blocking unary)
- sync_ch  = grpc.insecure_channel(...) → ExchangeGradients (called from training thread)
- Reason: training loop runs in run_in_executor(), can't use aio there

## Leader internals (trainer/leader/)
- asyncio.Lock (_lock): guards _workers dict
- asyncio.Condition (_grad_cond): gradient sync — last worker to arrive aggregates + notifies all
- asyncio.Event (_training_started): set when operator types 'start' or auto-start fires
- Per-worker asyncio.Queue (assignment_queue): start_training() puts sentinel, unblocks GetAssignment
- heartbeat_monitor(): removes timed-out workers, handles dead-worker mid-sync
- _grad_arrival dict + _prev_round_end: per-round latency tracking for straggler detection
- _count_alive(): only counts workers with assigned=True (late joiners excluded until assigned)
- LeaderCore mixin: all computation methods — model init, aggregation, sharding, validation
- LeaderService: gRPC handlers only, inherits LeaderCore

## Data flow per training step
1. Worker: forward pass → loss.backward() → compress_gradients() [Top-K per layer]
2. Worker: ExchangeGradients RPC → sends GradientPush (worker_id, step, loss, sparse_grads)
3. Leader: collects from all alive+assigned workers (asyncio.Condition wait loop)
4. Leader: last worker triggers _aggregate_and_step() → weighted average → optimizer.step()
   Computes per-worker compute_ms and wait_ms, flags straggler if wait > 200ms
5. Leader: serializes weight delta (new_weights - old_weights, float16) → WeightUpdate
6. Worker: apply_delta() adds delta to local model params in-place

## Gradient compression
Top-K sparsification: for each layer, keep topk_k elements by absolute value.
Sends (indices, values, shape) tuples. Default topk_k=50000. topk_k=0 = full gradients.
One device sync before loop (torch.mps.synchronize / torch.cuda.synchronize) to avoid
per-layer implicit syncs.

## Sharding
worker_fraction = worker_score / sum(all_scores)
worker_samples  = int(fraction × train_samples)
batch_size scales proportionally (base from --batch-size, default 32, min 8, max 256)
Phase-2 reshard: when a late worker is admitted, all active workers are resharded
proportionally and sent a RESHARD command via the next heartbeat response.

## RunRecorder output
runs/<model>__<dataset>__<optimizer>__topk<K>__<timestamp>/
  config.json     hyperparameters snapshot
  metrics.csv     columns: step, loss, round_ms, straggler_delay_s, epoch, val_acc
  summary.json    final_loss, final_val_acc, total_steps, compression_ratio, throughput
  loss.png        loss curve (requires matplotlib)

## Tailscale
Leader DNS: leader-macbook-pro.taila5426e.ts.net
ALWAYS use the Tailscale magic DNS — never 127.0.0.1 or localhost, even for local tests


# KEY DECISIONS & CONSTRAINTS

1. Parameter Server over DDP: MPS + CUDA can't use NCCL. PS handles heterogeneous hardware.

2. Two gRPC channels per worker (async + sync): training thread uses run_in_executor().

3. Weight delta (not full state dict) per step: reduces bandwidth from ~170 MB to ~1-10 MB.
   Delta serialized as float16 (halves size: 43 MB → 22 MB) with negligible precision loss.

4. asyncio.Condition for gradient sync: "last worker aggregates" semantics.

5. LeaderCore mixin: all computation methods in a mixin so self.* access patterns are
   unchanged — zero risk of attribute bugs when splitting the class.

6. Worker bootstrap (trainer/worker/bootstrap.py): stdlib-only, runs before third-party
   imports. Creates .venv, runs pip install -e ., generates proto stubs, re-execs inside
   venv via os.execv (Mac/Linux) or subprocess.run (Windows).
   NOTE: bootstrap is only used by scripts/ (one-shot fresh-clone setup). The normal
   path is `pip install -e .` which creates dtrain-* entry points directly.

7. Late worker joining: accepted mid-run, gets proportional random shard + current weights.
   _count_alive() excludes unassigned workers so they don't block existing grad rounds.

8. Auto-reconnect: leader_disconnected flag distinguishes gRPC failure from explicit STOP.
   On disconnect: sleep 10s → re-register with new leader.

9. Test isolation: tests use CPU-only fake hardware to avoid MPS Metal assertion crashes
   when two training threads call mps.synchronize() concurrently.

10. grad_sync_timeout (120s): handles extreme stalls when a worker dies mid-round.

# KNOWN ISSUES / RISKS

1. [FIXED] gRPC port conflict: duplicate leader processes on same port.
   Fixed: grpc.so_reuseport=0 + kill_existing_leader() on startup.

2. [FIXED] grpc.aio has no sleep() — heartbeat crashed. Fixed: asyncio.sleep(5).

3. [FIXED] gRPC 4 MB message limit. Fixed: 256 MB server options.

4. [FIXED] Leader exited in subprocess mode. Fixed: isatty() check.

5. [FIXED] Late worker rejection. Fixed: accepts mid-run, assigns shard + reshard.

6. [FIXED] Worker exits on leader restart. Fixed: auto-reconnect loop with 10s retry.

7. [FIXED] Test 3 MPS crash: two training threads calling mps.synchronize() concurrently.
   Fixed: test uses CPU-only fake hardware (protocol correctness, not GPU performance).

8. [FIXED] T11 test bug: queried worker status before start_training() admitted workers.
   Fixed: check both status.workers and status.pending_workers.

9. [RISK] ResNet-101 weight delta still ~44 MB per step on first exchange.
   Mitigation: TopK compression reduces gradient payload.

10. [KNOWN] Straggler detection logged but not mitigated.
    grad_sync_timeout (120s) prevents full stall. Per-worker latency in CSV for analysis.

11. [KNOWN] Windows bootstrap not yet tested end-to-end on real CUDA hardware.

12. [KNOWN] MPS + asyncio.run_in_executor may have restrictions in some PyTorch versions.
    If MPS training hangs, pin training thread to main thread.
