Distributed ResNet Training

Parameter-server training across heterogeneous laptops via Tailscale

ResNet-18 / 50 / 101 Tiny ImageNet-200 MPS + CUDA Tailscale VPN gRPC
1Start leader
2Join workers
3Type start
4Training runs
5Results saved

Step 1 — Start the Leader

Leader machine (parameter server) macOS 
# Navigate to the project and pull latest
cd /Users/ruturaj_vasant/Desktop/Academic/AI_New/distributed-resnet
git pull
python3 leader.py --model resnet18 --epochs 1 --lr 0.01 --batch-size 32
Wait for Commands: start | status | reset | quit — do not type start yet. Add workers first.

Step 2 — Join as a Worker

Leader machine — second terminal macOS 
cd /Users/ruturaj_vasant/Desktop/Academic/AI_New/distributed-resnet
python3 worker.py
Any other Mac — fresh clone macOS 
# Clone repo, bootstrap runs automatically (venv + deps + dataset)
git clone https://github.com/Ruturaj-Vasant/Distributed_Hetrogeneous_Training.git ~/distributed-resnet
cd ~/distributed-resnet
python3 worker.py
Any Windows machine — fresh clone Windows (PowerShell) 
git clone https://github.com/Ruturaj-Vasant/Distributed_Hetrogeneous_Training.git $HOME\distributed-resnet
cd $HOME\distributed-resnet
python worker.py
First run downloads PyTorch + deps (~1 GB) and Tiny ImageNet-200 (~236 MB). Subsequent runs skip this entirely. Add --dry-run to test the protocol with synthetic data only.
Add as many workers as you like. Each one self-registers, gets a data shard proportional to its hardware score, and starts training when you type start. Workers automatically reconnect if the leader restarts.

Step 3 — Begin Training

In the leader terminal — once all workers are registered 
start
Other interactive commands: status — print progress line  |  reset — clear state for next run  |  quit — stop leader (workers reconnect automatically)

Optional — Live Cluster Monitor

Any terminal on any machine in the Tailscale network macOS

Leader Flags Reference

Flag Default Description
--model resnet101 resnet18, resnet50, or resnet101
--epochs 90 Number of training epochs
--lr 0.1 SGD learning rate
--batch-size 32 Base batch size — each worker scales proportionally to its hardware score
--weight-decay 1e-4 SGD weight decay (L2 regularisation)
--topk 50000 Top-K gradient elements kept per layer. 0 = full gradients (no compression)
--port 50051 gRPC server port
--min-workers 1 Minimum workers before auto-start
--auto-start off Start automatically when min-workers threshold is met
--data-root ~/.cache/tiny-imagenet-200 Override dataset cache path
Worker Flags Reference
Flag Default Description
--leader leader-macbook-pro.taila5426e.ts.net Leader hostname or Tailscale magic DNS
--port 50051 Leader gRPC port
--dry-run off Use synthetic data — tests the protocol without downloading the dataset
--cache-dir ~/.cache/tiny-imagenet-200 Override dataset cache path

Architecture

  ┌─────────────────────────────────────┐
  │            Leader  (Mac)            │
  │  ┌───────────────────────────────┐  │
  │  │  Parameter Server (gRPC)      │  │
  │  │  • Aggregate gradients        │  │
  │  │  • SGD optimizer step         │  │
  │  │  • Send weight delta back     │  │
  │  │  • Val accuracy after epoch   │  │
  │  └───────────────────────────────┘  │
  └──────┬──────────┬───────────────────┘
         │          │   Tailscale VPN
  ┌──────┴──┐  ┌────┴──────┐
  │Worker 1 │  │ Worker 2  │  ← any number
  │Mac/MPS  │  │ Win/CUDA  │
  │ TopK ↑  │  │ TopK ↑    │
  └─────────┘  └───────────┘
  Each worker: forward → loss.backward() → Top-K compress → ExchangeGradients RPC
